Web Server & DNS Server

Web Server

  1. Create a socket
    • Set the server's socket port to port 80
    • Set the server's ip to local ip
    • Set the server's buffer size to something small
    • Listen for as many connections as you want to serve (we chose 1)
  2. Accept incoming traffic
    • Grab IP address of user from the data they provide when trying to connect to the webserver
  3. Return blacksite web page as response
  4. Print out response
  5. Close connection w/ server

DNS Server

  1. Create a socket
    • Set the server's socket port to port 53
    • Set the server's ip to local ip
    • Set the server's buffer size to something small
    • Listen for as many connections as you want to serve (we chose 1)
  2. Accept incoming traffic
  3. Generate DNS traffic
    • Grab transaction ID of request packet
    • Set flags to pre-set defaults for our packet (except for opcode which is grabbed from the packet)
    • Grab question, answer, nameserver, and additional count of request packet
    • Get domain name from request packet
    • Create DNS question and body with all of this information.
  4. Return DNS response to the client address

Packet Structure

  1. Preamble
    • 11111111
  2. Header
    • Packet ID: 00000000
    • Length: 00000000
  3. Body
    • Data: 00000000
  4. Error Detection
    • Bit Parity: 00000000

Encryption

One Time Pad

  1. Randomly Generate Key that is exactly as long as data being transmitted
    • We just used 10101010
    • When converted to an integer it is 170
    • Length is 8 bits because the length of an ASCII character is 8 bits
    • Also this is easier than dynamically generating random keys and having to manually code them into the decryptor
      • That is how a true one time pad would work though.
  2. Encrypt information with key
    • We OR 170 with the original data to produce encrypted data
  3. Decrypt information with key
    • We OR 170 with the encrypted data to produce original data
    • This works because of the mathematical property of OR
